01 / PRINCIPLE

노드를 고르기 전, 먼저 잡을 5가지 원리

이 5가지만 머릿속에 있으면 “왜 안 되지?”의 90%가 사라지고, 어떤 노드를 어디에 둘지 감이 잡힙니다.

데이터는 “표(데이터셋)”로 흐른다

거의 모든 노드는 행과 열로 된 표(데이터셋)를 주고받습니다. 텍스트 한 줄도, PDF도 결국 표로 바뀌어 다음 노드로 넘어갑니다. “지금 이 선을 타고 어떤 표가 흐르나?”를 늘 떠올리세요.

포트 색(타입)이 맞아야 연결된다

노드의 점(포트)에는 종류가 있습니다 — 데이터셋 · 모델 · 페이지 등. 같은 종류끼리만 선이 이어집니다. 연결이 안 되면 십중팔구 포트 타입이 다른 것입니다.

프롬프트는 “행마다 한 번씩” 실행된다

프롬프트 노드는 표의 각 행에 대해 AI를 1번씩 돌리고, 결과를 새 열로 붙입니다. 행이 100개면 AI 호출도 100번. 글 속 값은 {{컬럼명}}으로 꽂습니다.

비용(크레딧) = 행 수 × 모델

프롬프트는 행 수 × 모델 단가만큼 크레딧을 씁니다. 처음엔 데이터 분할로 몇 행만 떼어 테스트하고, 잘 되면 전체에 돌리세요. 모델도 가벼운 것(gpt-4o-mini)부터.

모든 흐름은 4단계다

① 가져오기 → ② 다듬기 → ③ AI 처리 → ④ 내보내기. 어떤 자동화든 이 뼈대입니다. 만들기 전에 “내 일을 이 4칸에 적으면 뭐지?”부터 종이에 써보세요.

03 / CORE

이것부터 익히면 되는 핵심 노드 20

144개 중 현업 워크플로우의 80%는 아래 노드들로 만들어집니다. 입력→출력은 어떤 포트가 들어오고 나가는지를 뜻합니다.

프롬프트

표의 각 행을 AI에게 보내 요약·분류·작성·추출하고, 결과를 새 열로 받습니다. 가장 핵심.

입력 데이터셋 · 출력 데이터셋(+AI 응답 열)

언제? “이 내용을 AI가 처리해줬으면” 싶은 거의 모든 순간. 글 안에 {{컬럼명}}으로 값 꽂기.

에이전트 프롬프트

프롬프트 + 웹검색·계산·조회 같은 도구를 AI가 스스로 사용. 단순 처리 이상이 필요할 때.

입력 데이터셋 · 출력 데이터셋

언제? 최신 정보 검색, 자료 조회, 여러 단계 판단이 필요한 챗봇·자동화.

텍스트 입력

사용자가 직접 텍스트를 넣는 출발점. 가장 단순한 입력 노드.

입력 없음 · 출력 텍스트/데이터셋

언제? 테스트하거나, 고정된 내용을 흘려보낼 때. (표가 필요하면 텍스트-데이터 변환 연결)

데이터 / 데이터 테이블

CSV·엑셀 등 표 데이터를 올리거나, 표를 직접 보고 편집하는 노드.

입력 (없음/데이터셋) · 출력 데이터셋

언제? 가진 표 파일로 시작할 때. 중간 점검용으로도 자주 연결.

PDF 데이터 → PDF-데이터 변환

PDF를 올리고(앞), 그 안의 텍스트를 표로 추출(뒤). 둘은 보통 짝으로 씁니다.

입력 없음→pdf · 출력 pdf→데이터셋

언제? 보고서·매뉴얼·계약서 등 PDF 내용을 AI로 처리할 때 첫 두 칸.

데이터 조건 분기

데이터 값에 따라 흐름을 갈라 다른 작업을 실행. if/else 역할.

입력 데이터셋 · 출력 데이터셋(조건별)

언제? “긴급이면 슬랙, 아니면 메일”처럼 결과에 따라 다르게 처리할 때.

데이터 분할

표를 일부/나머지로 나눕니다. 테스트용 샘플을 떼는 데 유용.

입력 데이터셋 · 출력 샘플링된/남은 데이터셋

언제? 프롬프트를 전체에 돌리기 전 몇 행만 먼저 테스트해 크레딧 아끼기.

크롤링 / 고급 크롤링

네이버·구글 검색 등에서 데이터를 자동 수집해 표로 만듭니다.

입력 (키워드 설정) · 출력 데이터셋

언제? 뉴스·검색 결과·웹 정보를 정기적으로 모아올 때.

구글 스프레드시트

구글 시트를 읽어와 표로 사용. (Gmail·드라이브·캘린더·폼도 같은 방식)

입력 없음 · 출력 데이터셋

언제? 이미 시트로 관리하는 데이터를 자동화에 끌어올 때. 최초 1회 구글 연동 필요.

유튜브 요약

영상 링크를 넣으면 자막 기반으로 핵심을 정리. (검색·댓글 노드와 함께)

입력 (URL) · 출력 데이터셋

언제? 경쟁사/시장 영상, 강연을 빠르게 인사이트로 만들 때.

열 선택 / 행 선택

필요한 열만 고르거나, 조건에 맞는 행만 남깁니다. 표 정리의 기본기.

입력 데이터셋 · 출력 데이터셋

언제? AI에 넘기기 전 불필요한 열을 쳐내 정확도·비용을 챙길 때.

데이터 병합 / 그룹화

두 표를 키로 합치거나(병합), 특정 열 기준으로 묶어 집계(그룹화).

입력 데이터셋(2개) · 출력 데이터셋

언제? 여러 소스를 합쳐 보거나, “부서별 합계”처럼 요약할 때.

텍스트 임베딩

텍스트를 숫자 벡터로 바꿔 의미 검색이 가능하게. RAG의 핵심 재료.

입력 데이터셋 · 출력 데이터셋(벡터)

언제? 문서가 많아 “관련 부분만 찾아 답하게” 하고 싶은 RAG 챗봇.

음성-텍스트 변환

녹음/음성 파일을 텍스트로 받아쓰기. 회의·인터뷰 처리에 유용.

입력 오디오 · 출력 데이터셋(텍스트)

언제? 회의 녹음 → 받아쓰기 → 프롬프트로 요약/액션아이템.

이메일 전송

결과를 메일로 자동 발송. 제목·본문에 {{컬럼명}}·{{data}} 변수 사용.

입력 데이터셋(선택) · 출력 없음

언제? 자동화 마지막 단계. {{컬럼명}}은 첫 행, {{data}}는 표 전체.

슬랙 / 텔레그램 전송

결과를 메신저로 발송. 텔레그램 트리거로 “메시지 받으면 실행”도 가능.

입력 데이터셋 · 출력 없음

언제? 팀 채널 알림, 봇처럼 받고-답하는 흐름.

챗 UI

사용자가 대화로 입력하고 AI 답을 받는 채팅 화면. 챗봇 앱의 얼굴.

입력 데이터셋/에이전트 · 출력 페이지

언제? 사내 Q&A 봇, FAQ 봇처럼 대화형 앱을 만들 때.

데이터 테이블 / 텍스트 출력

결과 표나 텍스트를 앱 화면에 보여주는 출력 UI.

입력 데이터셋 · 출력 페이지

언제? 대시보드·리포트 형태로 결과를 보여줄 때.

막대 / 선 / 원 그래프

표를 차트로. X축·Y축·집계함수(평균·합계 등)만 고르면 끝.

입력 데이터셋 · 출력 없음(차트)

언제? 수치 추이·비중·비교를 한눈에 보여줄 때. X축 선택은 필수.

배포

완성한 화면(페이지)을 실제 웹앱으로 게시해 동료가 쓰게 합니다.

입력 페이지/배포 · 출력 없음(URL 생성)

언제? 워크플로우를 “쓸 수 있는 앱”으로 공유하는 마지막 단계.

04 / RECIPE

그대로 따라 만드는 워크플로우 레시피 5

아래 순서대로 노드를 추가하고 선으로 이으면 됩니다. 색 = 카테고리(찾을 위치). 본인 업무에 맞게 입력 노드만 바꿔보세요.

① 표 데이터에 AI를 한 번에 돌리기

고객 문의·리뷰·리스트 등 표의 각 행을 AI로 분류/요약/답변 초안 만들기

구글 스프레드시트→ 열 선택→ 프롬프트→ 데이터 테이블
프롬프트엔 “다음 문의를 환불/배송/기타로 분류해줘: {{문의내용}}”처럼 작성. 먼저 데이터 분할로 5행만 테스트 후 전체 실행.

② 뉴스·검색 모니터링 자동 보고

키워드 수집 → AI 요약/중요도 판단 → 조건에 맞으면 메일·슬랙으로 보고 (스케줄로 매일 자동)

크롤링→ 프롬프트→ 데이터 조건 분기→ 이메일 전송
프롬프트로 “요약 + 중요도(상/중/하)” 열을 만들고, 조건 분기로 ‘상’만 통과시켜 메일 발송. 완성 후 캔버스 ‘스케줄 설정’으로 매일 아침 자동 실행.

③ 사내 문서 Q&A 챗봇 (RAG)

매뉴얼·규정 PDF를 근거로 질문에 답하는 대화형 앱

PDF 데이터→ PDF-데이터 변환→ 텍스트 임베딩→ 에이전트 프롬프트→ 챗 UI
임베딩~검색 연결의 정확한 구성은 docs의 ‘사내 문서 RAG 챗봇’ 시나리오를 참고하세요. 프롬프트엔 “문서에 없으면 모른다고 답하라”를 꼭 넣어 환각을 줄입니다.

④ 회의 녹음 → 요약·액션아이템

녹음 파일을 받아쓰고 AI가 담당자·할 일·기한으로 정리

음성 녹음→ 음성-텍스트 변환→ 프롬프트→ 슬랙 메시지 보내기
프롬프트 형식을 “담당자 · 할 일 · 기한 표로”라고 지정하면 정리가 깔끔합니다. 결과를 팀 채널로 자동 공유.

⑤ 결과를 웹앱으로 배포

만든 워크플로우를 동료가 직접 쓸 수 있는 화면으로 공개

(앞의 처리 흐름)→ 데이터 테이블 / 챗 UI→ 페이지→ 배포
UI 노드들을 페이지에 담아 화면을 구성하고 배포로 게시하면 URL이 생깁니다. 표만 외부에 넘기려면 데이터셋 API 배포를 사용.
04.5 / TROUBLESHOOT

“왜 안 되지?” 자주 막히는 7가지

현업이 가장 많이 멈추는 지점과 해결법입니다. 막히면 여기부터 보세요.

선이 연결이 안 돼요
포트 타입이 다른 것입니다. 데이터셋 출력은 데이터셋 입력에만 꽂힙니다. 점 색/이름을 맞춰보세요.
프롬프트에서 변수가 안 들어가요
{{컬럼명}}의 이름이 실제 열 이름과 정확히 일치해야 합니다. 띄어쓰기·오타가 대부분의 원인.
텍스트를 프롬프트에 못 넣어요
프롬프트 입력은 데이터셋(표)입니다. 일반 텍스트는 텍스트-데이터 변환을 거쳐 표로 바꿔 연결하세요.
크레딧이 너무 빨리 닳아요
프롬프트는 행마다 실행됩니다. 데이터 분할로 일부만 테스트하고, 가벼운 모델부터 쓰세요.
